# Testing Guidelines

Testing in ZTWIM follows a two-tier architecture: fast unit tests under `pkg/` and integration-level E2E tests under `test/e2e/` that run against a live OpenShift cluster.

## Make Targets

| Command | What it does |
|---------|-------------|
| `make test` | Unit tests via envtest (K8s 1.31.0 binaries). Sets `OPERATOR_NAMESPACE`. Excludes `test/e2e/`. |
| `make test-e2e` | E2E tests against a live cluster. Requires `KUBECONFIG`. Default timeout 45 min. |
| `make verify` | `vet` + `fmt` + `golangci-lint` (includes `ginkgolinter`). |
| `make lint` | `golangci-lint` only. |

Unit tests use `setup-envtest` with `ENVTEST_K8S_VERSION = 1.31.0`. The binary is downloaded to `bin/` on first run.

## Unit Tests (`pkg/controller/`)

### File Layout

Each reconciler sub-function gets its own test file:

```
pkg/controller/spire-server/
├── controller_test.go          # Reconcile() entry point tests
├── service_account_test.go     # reconcileServiceAccount tests
├── service_test.go             # reconcileService tests
├── rbac_test.go                # reconcileRBAC tests
├── configmaps_test.go          # reconcileConfigMap tests
├── statefulset_test.go         # reconcileStatefulSet tests
├── routes_test.go              # reconcileRoute tests
├── webhook_test.go             # reconcileWebhook tests
└── validation_test.go          # validateConfiguration tests
```

### Test Reconciler Factory

Every controller package defines a `newTestReconciler` helper that wires up the fake client, a discard logger, a fresh scheme, and a fake event recorder:

```go
func newTestReconciler(fakeClient *fakes.FakeCustomCtrlClient) *SpireServerReconciler {
    return &SpireServerReconciler{
        ctrlClient:    fakeClient,
        ctx:           context.Background(),
        log:           logr.Discard(),
        scheme:        runtime.NewScheme(),
        eventRecorder: record.NewFakeRecorder(100),
    }
}
```

When a test needs owner-reference or controller-ref logic, register types in the scheme:

```go
scheme := runtime.NewScheme()
_ = v1alpha1.AddToScheme(scheme)
_ = corev1.AddToScheme(scheme)
reconciler.scheme = scheme
```

### FakeCustomCtrlClient

Generated by counterfeiter (`pkg/client/fakes/`). Never hand-edit. Regenerate with:

```bash
go generate ./pkg/client/...
```

Key stubbing patterns:

| Pattern | Use case |
|---------|----------|
| `fakeClient.GetReturns(err)` | Every `Get` call returns the same error. |
| `fakeClient.GetStub = func(...)` | Different behaviour per call (switch on call count or object type). |
| `fakeClient.CreateReturns(nil)` | All creates succeed. |
| `fakeClient.UpdateReturns(err)` | Simulate update conflict. |
| `fakeClient.StatusUpdateWithRetryReturns(nil)` | Status writes succeed. |
| `fakeClient.CreateCallCount()` | Assert a Create was invoked. |
| `fakeClient.UpdateCallCount()` | Assert an Update was invoked. |

For multi-step reconcilers, use `GetStub` with a call counter or type-switch:

```go
fakeClient.GetStub = func(ctx context.Context, key client.ObjectKey, obj client.Object) error {
    switch v := obj.(type) {
    case *v1alpha1.SpireServer:
        *v = *spireServer
        return nil
    case *v1alpha1.ZeroTrustWorkloadIdentityManager:
        *v = *ztwim
        return nil
    default:
        return kerrors.NewNotFound(schema.GroupResource{}, key.Name)
    }
}
```

### Table-Driven Test Pattern

All scenario sets use `[]struct` + `t.Run`:

```go
tests := []struct {
    name        string
    setupClient func(*fakes.FakeCustomCtrlClient)
    expectError bool
}{...}

for _, tt := range tests {
    t.Run(tt.name, func(t *testing.T) {
        fakeClient := &fakes.FakeCustomCtrlClient{}
        reconciler := newTestReconciler(fakeClient)
        tt.setupClient(fakeClient)
        // ... exercise + assert
    })
}
```

### Required Test Scenarios Per Sub-Function

Every `reconcileX` function must cover:

1. **Create success** — `Get` returns `NotFound`, `Create` returns nil.
2. **Create error** — `Get` returns `NotFound`, `Create` returns error.
3. **Get error** — `Get` returns non-`NotFound` error.
4. **Update success** — resource exists with drift, `Update` returns nil.
5. **Update error** — resource exists with drift, `Update` returns error.
6. **Create-only mode skip** — resource exists, `createOnlyMode=true`, no Update.
7. **SetControllerReference error** — empty scheme causes ref failure.

### Environment Variables in Tests

Use `t.Setenv` (auto-cleaned by `testing`):

```go
t.Setenv("CREATE_ONLY_MODE", "true")
t.Setenv("OPERATOR_NAMESPACE", "zero-trust-workload-identity-manager")
```

### Status Manager in Tests

```go
statusMgr := status.NewManager(fakeClient)
err := reconciler.reconcileServiceAccount(ctx, server, statusMgr, createOnlyMode)
```

## E2E Tests (`test/e2e/`)

### Framework

- **Ginkgo v2** + **Gomega** (dot-imported).
- Suite file: `e2e_suite_test.go` — sets up `k8sClient`, `clientset`, `apiextClient`, `configClient`.
- Specs: `e2e_test.go` — single `Describe("Zero Trust Workload Identity Manager", Ordered, ...)`.

### Configuration

| Setting | Value |
|---------|-------|
| Suite timeout | 40 min |
| `go test -timeout` | 45 min (via `E2E_TIMEOUT`) |
| FailFast | `false` |
| Verbose | `true` |
| JUnit report | `$ARTIFACT_DIR/junit.xml` (Prow) or `/tmp/junit.xml` |

### Timeouts (from `test/e2e/utils/constants.go`)

| Constant | Duration | Use |
|----------|----------|-----|
| `DefaultTimeout` | 5 min | Standard Eventually waits |
| `ShortTimeout` | 2 min | CRD established, deployment available |
| `TestContextTimeout` | 10 min | Per-spec context deadline |
| `DefaultInterval` | 10 sec | Polling for deployments/statefulsets |
| `ShortInterval` | 5 sec | Polling for conditions/CRDs |

### Writing a New E2E Test

1. Add an `It(...)` block inside the appropriate `Context(...)` in `e2e_test.go`.
2. Use `testCtx` (per-spec context with 10-min deadline) for all API calls.
3. Use helpers from `test/e2e/utils/`:
   - `WaitForStatefulSetReady`, `WaitForDaemonSetAvailable`, `WaitForDeploymentAvailable`
   - `WaitForSpireServerConditions`, `WaitForSpireAgentConditions`, etc.
   - `SetupAttestationTest` — creates isolated namespace + ClusterSPIFFEID + pod with CSI.
   - `UpdateCRWithRetry` — conflict-safe update helper.
4. Cleanup via `DeferCleanup(func(ctx context.Context) { ... })`.
5. Use `fmt.Fprintf(GinkgoWriter, ...)` for debug output (appears only on failure/verbose).

### E2E Test Structure Example

```go
It("should reconcile custom labels on SpireServer pods", func() {
    By("Updating SpireServer CR with custom labels")
    err := utils.UpdateCRWithRetry(testCtx, k8sClient, spireServer, func() {
        spireServer.Spec.CommonConfig.Labels = map[string]string{"team": "security"}
    })
    Expect(err).NotTo(HaveOccurred())

    By("Waiting for StatefulSet rolling update")
    utils.WaitForStatefulSetRollingUpdate(testCtx, clientset,
        utils.SpireServerStatefulSetName, utils.OperatorNamespace, initialGen, utils.DefaultTimeout)

    By("Verifying pod labels")
    pods, err := clientset.CoreV1().Pods(utils.OperatorNamespace).List(testCtx,
        metav1.ListOptions{LabelSelector: utils.SpireServerPodLabel})
    Expect(err).NotTo(HaveOccurred())
    utils.VerifyPodLabels(utils.FilterActivePods(pods.Items), map[string]string{"team": "security"})
})
```

### Running E2E Locally

```bash
export KUBECONFIG=~/.kube/config
make test-e2e                           # full suite
go test ./test/e2e/ -v -timeout 45m -run "Installation"  # single context
```

## Adding a New Unit Test (Checklist)

1. Create `pkg/controller/<component>/<function>_test.go`.
2. Use the package's `newTestReconciler` (or define `newXXXTestReconciler` if the scheme differs).
3. Define a `tests` slice with `name`, `setupClient`, `expectError`, and any `expect*` assertion fields.
4. Instantiate `&fakes.FakeCustomCtrlClient{}` per sub-test.
5. Stub `Get`/`Create`/`Update` as needed.
6. Call the function under test with a fresh `status.NewManager(fakeClient)`.
7. Assert error/no-error and call counts (`CreateCallCount`, `UpdateCallCount`).
8. Run: `make test` or `go test ./pkg/controller/<component>/...`.

## Linting for Tests

The `.golangci.yml` enables `ginkgolinter` which enforces Gomega assertion style in Ginkgo tests. The `dupl` linter is suppressed under `pkg/` to allow repetitive table-driven test scaffolding.
